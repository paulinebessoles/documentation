= Propuestas

== Understand the basics

The Proposals component allows people to create, comment, like, and give their votes to proposals. It's one of the main components of
Decidim, and has a large number of configuration settings.

image::components/proposals/example01.png[Proposals example]

== Configuración

To create a Proposals component, first go in the xref:admin:spaces.adoc[participatory space] where you want to create it.

image::components/proposals/component_proposals.png[Add Proposals component button]

Then:

. Click on the "Components" item in the secondary menu
. Click the "Add component" button
. Select "Proposals"
. Fill the component creation form

You can still update this form and the configuration after creating the component, by clicking the
image:icons/action_configure.png[Configure icon] Configure icon in the components list.

The component creation form is divided into different sections detailed below.

image::components/proposals/component_01.png[Add component form]

.Añadir componente: Formulario de propuestas
[cols="20h,10h,~"]
|===
|Campo |Tipo |Descripción

|Nombre
|Requerido
|What is the title of this component.

|Posición del pedido
|Opcional
|The order of the component in relation with the other components. Tener un número más bajo significa una prioridad mayor.
For instance, if you have two components, "Component a" with weight "10" and "Component b" with weight "0"
then "component b" is first when ordering.
|===

image::components/proposals/component_02.png[Add component form]

.Añadir componente: Formulario de propuestas - Configuración global
[cols="20h,10h,~"]
|===
|Campo |Tipo |Descripción

|Ámbitos habilitados
|Opcional
|Marque si desea tener filtrado de Scopes en este componente.

|Ámbitos habilitados
|Opcional
|Check if you want to have Scopes filtering available in this component.

|Alcance
|Opcional
|Which xref:admin:settings/scopes.adoc[Scope] does this component belongs to.

|Vote limit per participant
|Opcional
|Debe ser un número. The maximum number of votes a participant can give to proposals.

|Minimum votes per user
|Opcional
|Debe ser un número. The minimum number of votes a participant needs to give to proposals so their votes are taken into account.

|Límite de propuesta por participante
|Opcional
|Debe ser un número. How many proposals a participant can create in this component.

|Longitud máxima del cuerpo de la propuesta
|Opcional
|Debe ser un número. Cuántos caracteres puede tener un cuerpo de propuesta como máximo.

|Edición de propuestas
|Requerido
|Choose if participants have a limited time to edit their proposals, or can edit them without limit.
In case it's within a specific time frame, you must specify the time in the field below.
Note that if an administrator xref:components/proposals/answers.adoc[answers a proposal] then it can't be edited anymore.

|Las propuestas pueden ser editadas por los autores antes de que pasen muchos minutos
|Requerido
|Debe ser un número. Cuántos minutos después de la creación de una propuesta puede ser editado por su autor.

|Umbral por propuesta
|Opcional
|Debe ser un número. How many votes a proposal needs to have to be considered as accepted.

|Can accumulate votes beyond threshold
|Opcional.
|Whether the proposal can accumulate more votes than the threshold.

|Respuesta de propuesta habilitada
|Opcional
|Si los administradores o los evaluadores pueden responder a las propuestas.

|Default proposal sorting
|Opcional
|The sorting order of the proposals in the proposals list. Automatic means that if the votes are enabled, the proposals are
shown sorted by random, and if the votes are blocked, then they are sorted by the most voted.

|Propuestas oficiales habilitadas
|Opcional
|Whether administrators can create official proposals from the administration panel.

|Comentarios habilitados
|Opcional
|Whether you want to enable participants to comment the proposals.

|Longitud máxima de los comentarios
|Opcional
|The characters limit that participants have when making comments. Deja 0 para el valor predeterminado.

|Geocodificación habilitada
|Opcional
|Whether participants can define a geocoding address for their proposals.
You need to have xref:configure:geocoding[Geocoding] enabled.

|Permitir archivos adjuntos
|Opcional
|Marcar para permitir a los participantes adjuntar archivos a sus propuestas. By enabling this option, proposals are by default in grid mode,
and the first image appears on the card.

|Se pueden establecer permisos de acciones para cada propuesta
|Opcional
|Whether you want to be able to set permissions for every proposals.

|Borradores colaborativos habilitados
|Opcional
|Check to enable xref:components/proposals/special_configurations/collaborative_drafts.adoc[collaborative drafts] in proposals.

|Textos participativos habilitados
|Opcional
|Check to enable xref:components/proposals/special_configurations/participatory_texts.adoc[participatory texts] in proposals.

|Enmiendas habilitadas
|Opcional
|Check to enable xref:components/proposals/special_configurations/amendments.adoc[amendments] in proposals.
If active, configure Amendment settings for each step.

|El Asistente de Enmiendas ayuda al texto
|Opcional.
|The help text visible when creating new amendments.

|Anuncio
|Opcional
|A general announcement visible on the proposals list page.

|Nueva plantilla de cuerpo de propuesta
|Opcional
|A pre-defined text used when creating new proposals. Útil si quieres definir un formato que los participantes necesitan
seguir al crear nuevas propuestas.

|Nuevo texto de ayuda de propuesta
|Opcional
|The help text visible when creating new proposals.

|Asistente de propuesta "Crear" texto de ayuda del paso
|Opcional
|The help text visible when creating new proposals in the "Create" step.

|Asistente de propuesta "Publicar" texto de ayuda del paso
|Opcional
|The help text visible when creating new proposals in the "Publish" step.
|===
