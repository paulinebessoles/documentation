= Participatory texts

== Understand the basics

This feature allows participants to navigate in the proposals like in a text, rather than a list or a grid. 
It can be useful for discussing norms, plans, or other kinds of texts.

Every paragraph is split into a proposal, so it's possible to discuss it in a more granular way.

image::components/proposals/participatory_texts/example01.png[Participatory text example]

== Configuration

To configure "Participatory texts" in proposals, enable the "Participatory texts enabled" setting in the component configuration form. 

image::components/proposals/component_specific_conf.png[Add component form - Specific configurations]

Click in the "Participatory texts" button on the top of the empty proposals component. 

image::components/proposals/participatory_texts/manage_proposals.png[Manage proposals]

Then, click in the "Import document" button in the empty Preview participatory text panel, to be redirected on the form. 

image::components/proposals/participatory_texts/add_document_form.png[Add document form]

.Add document form
|===
|Field |Type |Description

|Title
|Required
|The title of the document to be discussed in the proposal component. 

|Description
|Optional
|Description for the document.

|Document
|Required
|File. Add a document lesser than 2 MB, each section until 3 heading levels deep is parsed into proposals. 
Supported formats are: Markdown and ODT. 

|===

TIP: Although it's possible to work with both Markdown (.MD) and ODT (.ODT) files, we recommend working with Markdown as it's
easier to see the formatting. link:{attachmentsdir}/participatory_text.md[Download the example MD participatory text] for help.

Click the "Upload document" button to see the participatory text preview. 

== Preview the document

image::components/proposals/participatory_texts/preview_participatory_text.png[Preview participatory text]

Thanks to the preview, you can review if the document is correctly imported, and divided as you want it to be. 

Depending on the preview, you can:

* Discard: if you're not satisfied and want to reformat your document to start the import again. 
* Save draft: if you're satisfied with the result, but still wants to make some modifications. 
* Publish document: if you're ready to see how the text is like in the user interface. 

Once you published the document, all sections are presented as proposals in the proposal management panel. 

image::components/proposals/participatory_texts/imported_document_proposals.png[Imported document proposals]

Finally, you can publish the proposal component, for the participatory text to be visible by participants. 

NOTE: This feature could break existing proposals, so if there is any proposal it can't work, and a warning message is displayed:

====
Cannot interact with this setting if there are existing proposals. Please, create a new `Proposals component` if you want
to enable this feature or discard all imported proposals in the `Participatory Texts` menu if you want to disable it.
====
