= Page d'accueil

== En bref

Chaque processus et assemblage a une page de destination, avec différents blocs de contenu qui peuvent être ajoutés, supprimé, activé, ordonné et configuré,
comme xref:admin:settings/homepage.

Pour organiser une page de destination, entrez l'espace et cliquez sur l'élément de la page d'accueil dans la barre de navigation de gauche.
Vous êtes redirigé vers le panneau de configuration de la page d'accueil.

. Ajoutez différents blocs de contenu en cliquant sur le bouton « Ajouter un bloc de contenu » et choisissez celui que vous souhaitez ajouter.
. Configurer le bloc de contenu quand c'est possible.
. Enfin, déplacez le bloc de contenu dans la colonne des blocs de contenu actifs pour le faire apparaître sur la page d'accueil.

image::spaces/landing_page/landing_page_admin.png[Landing page content blocks]

== Configuration

Ci-dessous, vous trouverez tous les contenus de la page d'accueil qui bloquent les guides de configuration et les captures d'écran de la façon dont ils sont affichés sur la page d'accueil.

=== Image principale et bouton d'action

Les blocs d'image héros et de contenu CTA permettent d'afficher dans la page de destination de l'espace:

* *Titre* : le titre de l'espace.
* *Hashtag* : le hashtag de l'espace redirigeant vers la plate-forme X.
* *Sous-titre* : le sous-titre de l'espace.
* *Un bouton d'appel à l'action*: par défaut, le bouton n'est pas affiché. Pour l'activer, cliquez sur l'icône "Modifier" du bloc de contenu.
* *Une image* : l'image téléchargée est affichée avec un filtre d'opacité, pour s'assurer que le texte est lisible.

=== Annonce

Le bloc annonce affiche le contenu du champ d'annonce de la configuration de l'espace, dans une boîte bleue d'annonce.
Il permet aux administrateurs de mettre en valeur des informations importantes.

image::spaces/landing_page/announcement_front.png[Landing page content Announcement content block]

=== Données principales

Le bloc de données principal affiche la description courte et la description de l'espace et pour les assemblages, le but de l'action et de la composition.

image::spaces/landing_page/main_data.png[Landing page content Main data content block]

=== Phase / Type & durée

Le bloc phase & durée affiche la date de début et de fin du processus, et le bloc actuel xref:admin:spaces/processes/phases.adoc[phase] des processus.
Pour les assemblages, il affiche la durée de l'assemblage.

image::spaces/landing_page/duration.png[Landing page content Phase and duration content block]

=== Métadonnées

Le bloc de métadonnées affiche tous les champs de métadonnées saisis dans la configuration de l'espace.

image::spaces/landing_page/metadata.png[Landing page content Metadata content block]

=== Dernière activité

Le dernier bloc d'activité montre les dernières activités de l'espace, les articles de blog, les commentaires, les débats, etc.
Vous pouvez configurer le nombre d'avatars affichés à côté du titre.

image::spaces/landing_page/activities.png[Landing page content Last activity content block]

=== Documents connexes

Les documents connexes permettent à l'administrateur d'afficher les fichiers xref:admin:spaces/processes/attachments.adoc[Attachments] sur la page de destination, à l'exception des images.

image::spaces/landing_page/documents.png[Landing page content Related documents content block]

=== Photos associées

Les images associées permettent à l'administrateur d'afficher les images xref:admin:spaces/processes/attachments.adoc[Attachments] sur la page d'atterrissage.

image::spaces/landing_page/images.png[Landing page content Related images content block]

=== Réunions à venir

Les réunions à venir permettent à l'administrateur d'afficher le prochain xref:admin:components/meetings.adoc[Meetings] sur la page d'accueil avec une carte quand ils ont un emplacement.
Vous pouvez configurer le composant pour qu'il prenne les réunions en cliquant sur le bouton Modifier du bloc de contenu.

image::spaces/landing_page/meetings.png[Landing page content meetings content block]

=== Publications

Le bloc de contenu des messages permet à l'administrateur d'afficher les derniers messages publiés dans l'espace, avec leurs images si elles ont été ajoutées en tant que pièces jointes aux messages.
Vous pouvez configurer le composant pour prendre les messages en cliquant sur le bouton Modifier du bloc de contenu.

image::spaces/landing_page/posts.png[Landing page content posts content block]

=== Propositions

Les propositions permettent d'afficher un échantillon de propositions sur la page d'accueil de l'espace.

image::spaces/landing_page/proposals.png[Landing page content proposals block edition]

Vous pouvez configurer l'ordre d'affichage et le composant Proposition en cliquant sur le bouton Modifier du bloc de contenu.

image::spaces/landing_page/proposals_admin.png[Landing page proposals content block]

=== Résultats

image::spaces/landing_page/results.png[Landing page results content block]

Vous pouvez configurer l'ordre d'affichage et la responsabilisation pour prendre les résultats du composant en cliquant sur le bouton Modifier du bloc de contenu.

=== HTML

The HTML block content block allows to add an editable HTML block on the landing page, and therefore to create a personalized section.
