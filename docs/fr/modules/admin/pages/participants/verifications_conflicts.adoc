= Conflit de vérification

== En bref

Un conflit de vérification peut se produire lorsque les autorisations sont activées, et que certain·e·x·s participant·e·x·s créent plusieurs comptes
ou se font passer pour unx usurpateur·ice.

Par exemple :

. Unx participant·e·x crée un compte et se fait vérifier contre une autorisation.
. Puis crée un autre compte et tente de se faire vérifier à nouveau avec la même autorisation.

Cela peut également se produire avec des imitations :

. Unx participant·e·x obtient xref:admin:participants/impersonations.adoc[identification] avec ses données, mais ne saisie pas d'email
. Puis crée un compte et essaie de se vérifier avec la même vérification

Pour des raisons de sécurité, essayez d'être plus prudent dans ces cas, et assurez-vous qu'unx administrateur·ice examine ces conflits potentiels.
Cela dépend beaucoup du type de vérification que vous avez activé dans votre organisation, de sorte que la plupart du temps une organisation
ne voit aucun conflit apparaître dans ce panneau.

image:participants/verifications_conflicts.png[Panneau de conflits de vérification]

En cas de conflit de vérification, les administrateur·ices reçoivent une notification par mail.

== Configuration

=== Gérer les conflits de vérifications

Dans le panneau de vérification de conflit, vous pouvez trouver tous les conflits, avec :

* *Utilisateur* : nom de lae participant·e·x non géré·e·x qui a essayé d'être vérifié·e·x depuis son compte.
* *Utilisateur·ice géré·e·x* : nom de l'utilisateur·ice avec les mêmes données de vérification.
* *Tentatives* : nombre de fois où l'utilisateur·ice a essayé d'être vérifié·e·x avec ses données.
* *Résolu* : si le conflit de vérification a été résolu ou non.

Vous pouvez rechercher des conflits par mail, nom ou pseudonyme dans la barre de recherche.

=== Transférer l'utilisateur·ice

Le bouton de transfert utilisateur·ice permet de résoudre le conflit entre les deux comptes.

image:participants/verifications_conflicts_transfer_user.png[Formulaire de transfert d'utilisateur·ice]

.Formulaire de transfert utilisateur·ice
[cols="20h,10h,~"]
|===
|Champ |Type |Description

|Raison
|Obligatoire
|Pourquoi souhaitez-vous transférer le compte utilisateur·ice

|E-mail
|Obligatoire
|L'e-mail de l'utilisateur·ice à transférer
|===

Après le transfert de l'utilisateur·ice, les deux comptes, celui géré et le nouveau, sont fusionnés en un seul, qui prend l'adresse e-mail
saisie dans le formulaire, et la vérification validée est transférée sur ce compte.
