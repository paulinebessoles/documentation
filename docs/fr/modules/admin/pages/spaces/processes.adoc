= Processus participatifs

== En bref

Un processus participatif est une séquence d'activités participatives dans le but de définir et de prendre une décision sur un sujet spécifique.
Par exemple, vous pourriez avoir un processus participatif comme ceci :

. Répondre d'abord à une enquête pour recueillir l'opinion des participants ;
. Puis ouvrir des propositions;
. Discuter en face à face ou en réunion virtuelle ;
. Enfin en les priorisant avant leur mise en œuvre.

NOTE: Vous voulez créer un processus participatif ? Vérifiez l'article xref:admin:spaces/processes/process_creation.adoc[création de processus] .

Les exemples de processus participatifs sont : un processus d'élection des membres du comité (où les candidatures sont présentées pour la première fois,
alors débattu et finalement une candidature est choisie), des budgets participatifs (où des propositions sont faites, évalué économiquement,
et voté avec l'argent disponible), un processus de planification stratégique, l'élaboration collaborative d'un règlement ou d'une norme,
la conception d'un espace urbain ou la production d'un plan de politique publique.

Un concept vraiment important pour faire un bon usage de Decidim est de comprendre la relation entre les Phases et les Composants
dans un processus participatif de Decidim :

Ce que ce diagramme montre est que pour un processus participatif, vous pouvez configurer différentes phases, et activez et configurez
les composants pour chaque phase comme vous en avez besoin pour votre processus.

Dans chacune de ces Phases, vous pouvez activer xref:admin:components.adoc[Components] dont vous avez besoin.

=== How to find it?

Pour accéder au panneau des processus participatifs, allez dans le panneau d'administration et cliquez sur l'élément "Processus" dans la barre de navigation d'administration
.

image::spaces/processes/processes_menu.png[Pages: Access processes panel]

Là, vous pouvez gérer des processus et des groupes de processus participatifs.

== Configuration

=== Gérer les processus

image::spaces/processes/processes_list.png[Participatory processes list]

Sur cette page, vous pouvez gérer les processus et xref:admin:spaces/processes/groups.adoc[process groups].

Vous pouvez trouver tous les processus dans la liste dans le panneau d'administration, avec:

* *Titre* : titre du processus.
* *Créé à* : date de création du processus.
* *Privé* : si le processus est ouvert à tous les participants ou seulement à certains participants.
* *Publié* : si le processus est publié ou non.

Il est possible de filtrer et trier cette liste en cliquant sur "Filtrer" et sur les en-têtes des colonnes.
Vous pouvez également rechercher des processus par leur titre dans la barre de recherche.

image::spaces/processes/processes_list_filter.png[Participatory processes list filtered]

=== Importer un processus

Pour importer un processus, cliquez sur le bouton "Gérer" en haut à droite de la page.

image::spaces/processes/import_menu.png[Participatory processes import menu]

Vous êtes redirigé vers la page d'importation, où vous devez remplir ces champs :

* Titre : Le titre que vous souhaitez donner au processus.
* Identificateur d'URL : Le « slug » pour le processus.
* Ajouter un document : Ici vous devez charger le fichier JSON processus obtenu par e-mail lors de l'exportation.

Vous pouvez ensuite sélectionner le type de données que vous souhaitez importer à partir du fichier.

* Étapes d'import : Importe toutes les phases présentes dans le fichier original.
* Catégories d'importation : Importe toutes les catégories présentes dans le fichier original.
* Importer des pièces jointes : Importer des pièces jointes du processus original.
* Importer des composants : Importe tous les composants et leur configuration à partir du fichier original, sans importer leur contenu.
