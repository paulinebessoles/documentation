= Créer une concertation

== En bref

Les concertations permettent aux administrateur·ices de configurer des fonctionnalités en articulation avec différentes étapes participatives.
Pour créer une concertation, dans la liste des concertations, cliquer sur "Nouvelle concertation" en haut à droite de la page.

Le formulaire de création de concertation est divisé en différentes sections qui sont détaillées ci-dessous.

Une fois la concertation créée, dans le panneau de navigation secondaire, vous pouvez voir toute la configuration disponible :

* xref:admin:admin:spaces/processes/process_creation.adoc[À propos de cette concertation]: ici vous pouvez accéder et modifier tous les champs du formulaire de création.
* xref:admin:admin:spaces/processes/landing_page.adoc[Page d'accueil]
* xref:admin:spaces/processes/phases.adoc[Étapes]
* xref:admin:spaces/processes/components.adoc[Components]
* xref:admin:spaces/processes/categories.adoc[Categories]
* xref:admin:spaces/processes/attachments.adoc[Attachments]
* xref:admin:spaces/processes/admins.adoc[Administrateurs de la concertation]
* xref:admin:spaces/processes/private_participants.adoc[Utilisateurs privés]
* xref:admin:spaces/processes/moderations.adoc[Moderations]

== Configuration

=== Informations générales

Cette section permet de fournir une description générale de la concertation, principalement un titre, un URL et une description.

image::spaces/processes/processes_new_form_general_info.png[New participatory process form general info section]

.Formulaire pour créer une nouvelle concertation : Informations générales
[cols="20h,10h,~,~"]
|===
|Champ |Type |Description |Visibilité

|Titre
|Obligatoire
|Titre de la concertation. Par exemple, "Budget participatif 2020"
|Visible dans la carte de la concertation sur la page d'accueil, dans l'index des concertations, dans la navigation et dans le bloc de la page d'accueil "Image de bannière et CTA".

|Sous-titre
|Obligatoire
|Sous-titre. Par exemple, "décidons ensemble de nos priorités"
|Visible dans le bloc « Image de la bannière et CTA ».

|Rang d'affichage
|Obligatoire
|Détermine la position de la concertation lorsqu'il est affiché dans l'index des concertations ou la page d'accueil.
|N/A

|Identifiant d'URL
|Obligatoire
|Les identifiants d'URL sont utilisés pour générer les URL qui dirige vers ce processus. N'accepte que des lettres, des chiffres et des tirets et doit
commencer par une lettre.
|Visible dans l'URL de la concertation, par exemple https://example.org/processes/participatory-budget-2020.

|Hashtag
|Facultatif
|Hashtag redirigeant vers le réseau social X.
|Visible dans le bloc « Image de la bannière et CTA ».

|Obligatoire
|Une brève explication des sujets de la concertation.
|Visible dans le bloc "Données principales".

|Description
|Obligatoire
|Une explication détaillée des sujets de la concertation.
|Visible dans le bloc "Données principales".

|Annonce
|Facultatif
|Le texte entré ici est s'affiche juste en dessous des informations de processus, visible par les utilisateur·ices.
|Visible dans le bloc de la page d'accueil "Annonce".

=== Filtres

Cette section détermine les filtres disponibles pour la concertation, dans les différentes fonctionnalités.
Les filtres peuvent ensuite être activés et configurés dans chaque fonctionnalité.

image::spaces/processes/processes_new_form_filters.png[New participatory process form filters section]

.Formulaire pour créer une nouvelle concertation : Filtres
[cols="20h,10h,~"]
|===
|Champ |Type |Description

|Portées activées
|Facultatif
|Cocher cette case pour que le filtrage des secteurs soit effectué pour cette concertation.

|Secteur
|Facultatif
|À quel xref:admin:settings/scopes.adoc[Secteur] appartient cette concertation.

|Profondeur du filtre de secteur
|Facultatif
|Restreindre la profondeur de filtre du secteur. Ceci est pertinent si vous avez des relations complexes dans les Secteurs enfants (comme une poupée russe).
Par exemple, si vous avez des secteurs imbriqués Grand-mère -> Mère -> Enfant, ce paramètre vous permet de choisir le secteur Mère,
afin que les participant·e·x·s ne puissent filtrer qu'entre les sous-secteurs Enfant. Ce serait le cas par exemple si vous avez
Provinces -> Villes -> Districts, et que la concertation concerne une ville en particulier.

|Zone de stockage
|Facultatif
|À quelle xref:admin:settings/areas.adoc[Zone] appartient à cette concertation.
|===

|===

Il peut être activé dans le bloc de la page d'accueil "Concertations associées".

=== Autre

Cette section est vide, sauf si unx administrateur·ice a déjà créé xref:admin:spaces/processes/types.adoc[Types de concertation].

Les participant·e·x·s sont alors en mesure de filtrer les concertations dans l'index des concertations avec ces types.
