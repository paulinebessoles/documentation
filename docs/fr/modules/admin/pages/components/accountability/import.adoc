= Réalisations

== En bref

Parfois, le contenu de suivi est utilisé après un processus interne dans une organisation, telle l'évaluation du coût des projets de budget participatif dans des fichiers externes, il peut donc être utile d'importer de nouvelles réalisations et des modifications de réalisations par lots.

Pour cela, vous pouvez utiliser les fonctionnalités d'exportation et d'importation de réalisations.

== Configuration

=== Export de réalisations

Vous pouvez exporter les réalisations et leurs commentaires dans différents formats en cliquant sur « Tout exporter » dans la liste des réalisations :

* Réalisations au format CSV
* Réalisations au format JSON
* Réalisations au format Excel
* Commentaires en tant que CSV
* Commentaires en JSON
* Commentaires en tant qu'Excel

image::components/accountability/export.png[Import results from CSV form]

=== Réalisations

Vous pouvez également importer des réalisations en cliquant sur « Importer » dans la liste des réalisations du panneau d’administration.

image::components/accountability/import.png[Import results from CSV form]

==== Importer à partir d'une fonctionnalité

Vous pouvez importer les réalisations directement depuis une fonctionnalité xref:admin:components/budgets.adoc[Budget] du même espace participatif, et choisir d'importer tous les projets dans les réalisations ou seulement celle sélectionnés pour la mise en œuvre.

image:components/accountability/import_component.png[Importer les réalisations à partir d'un formulaire CSV]

==== Importer à partir d'un CSV

Vous pouvez aussi importer des réalisations à partir d'un fichier CSV.

image:components/accountability/import_csv.png[Importer les réalisations à partir d'un formulaire CSV]

Pour ce faire, suivre ces étapes :

. Créer les [Progrès] pour les Réalisations à ajouter
. Créer au moins une [Réalisation] manuellement à l'aide de ce panneau d'administration avant d'effectuer l'importation, afin d'avoir une meilleure compréhension du format et de ce que qu'il faut remplir.
. Télécharger l'[Export au format CSV]
. Effectuer les modifications localement. Il est seulement possible de modifier les colonnes CSV suivantes :
** category/id: ID pour la catégorie réalisation
** scope/id: ID pour l'objectif de la réalisation
** parent/id: ID du parent (pour les Réalisations liées). Facultatif
** title/en: Titre en français. Celui-ci dépend de la langue de configuration de la plateforme.
** description/en: Description en français. Celui-ci dépend de la langue de configuration de la plateforme.
** start_date: Date à laquelle la réalisation commence à s'exécuter (format AAAA-MM-JJ)
** end_date: Date à laquelle la réalisation termine l'exécution (format AAAA-MM-JJ)
** status/id: ID du Progrès de cette réalisation
** progress: Pourcentage (de 0 à 100) de l'exécution
** progress: ID interne des propositions associées (séparées par une virgule). Il est automatiquement converti en proposal_url
