= Gestion des propositions

== Understand the basics

Il est important de comprendre que les propositions ne sont pas seulement le nom du composant, mais aussi les entités du composant.

image::components/proposals/proposals_list_frontend.png[Proposals list frontend]

== Configuration

Une fois que vous avez créé votre volet Propositions, vous êtes redirigé vers la liste des propositions. Là, vous pouvez créer et gérer les propositions de
votre composant.

Selon que vous ne souhaitez réunir que des propositions de participants, ou que vous voulez également créer des propositions officielles, lisez la section
pertinente.

=== Création de propositions

==== Propositions officielles

Pour commencer à créer des propositions officielles, la première chose à faire est de créer votre première proposition, en cliquant sur le bouton "Nouvelle proposition".

==== Propositions des participants

Une fois que vous avez activé la création de propositions de participants pour la phase active, les participants peuvent voir un bouton "Nouvelle proposition" sur la page d'annonce de la proposition
.

image::components/proposals/new_proposal_button_frontend.png[New proposal button on proposals list]

Lorsqu'ils cliquent sur le bouton, ils sont redirigés sur un formulaire en deux étapes, leur permettant de créer leur proposition.
Ce formulaire dépend beaucoup des paramètres que vous avez activés dans le composant.

Une fois que la proposition est écrite et que toutes les propriétés sont définies, le participant peut cliquer sur le bouton d'aperçu pour examiner sa proposition.

Ils peuvent cliquer sur le bouton "Publier" pour finaliser la publication de la proposition.

=== Gérer les propositions

Une fois qu'il y a plusieurs propositions dans le processus, les administrateurs peuvent les gérer.

image::components/proposals/manage_proposals.png[Manage proposals table]

Sur la liste des propositions dans le panneau d'administration, les administrateurs peuvent voir :

* *Titre* : Titre de la proposition.
* *Publié à* : Date de publication de la proposition.
* *Commentaires* : Le nombre de commentaires que contient cette proposition.
* *Notes* : Le nombre de votes que cette proposition possède.
* *Valuators* : Le nombre d'évaluateurs pour cette proposition.
* *Statut* : Le statut de xref:admin:components/propositions/answers.adoc[answer] de cette proposition.

Vous pouvez filtrer et trier cette liste en cliquant sur le bouton "Filtre" et sur les en-têtes des colonnes.
Vous pouvez également rechercher des propositions par leur titre ou leur ID dans la barre de recherche.

* Propositions en JSON
* Propositions en tant qu'Excel
* Commentaires en tant que CSV
* Commentaires en JSON
* Commentaires en tant qu'Excel
