= Inscriptions

== En bref

NOTE: Cet article est dédié aux inscriptions "Sur cette plateforme" lors de la création ou la modification d'une réunion.

Grâce aux inscriptions, vous pouvez avoir le contrôle de la capacité de participation à une réunion. Avec cette fonctionnalité, par exemple, il est possible de
limiter le nombre de personnes pouvant assister à une réunion, ou de d'estimer s'il faut trouver une plus grande salle
pour tenir la réunion.

L'activation de cette fonctionnalité ajoute un bouton « S'inscrire » afin que les participant·e·x·s puissent exprimer leur souhait d'assister à la réunion.

image::components/meetings/registrations/frontend_join_meeting.png[Join a meeting button on meeting page]

Selon la façon dont cette fonctionnalité est configurée, les administrateur·ices peuvent :

. définir le nombre de créneaux disponibles pour contrôler la capacité maximale de cette réunion
. créer un xref:_registration_form[formulaire d'inscription personnalisé] pour demander des informations aux participant·e·x·s
. envoyer des [invitations] à d'autres participants ou personnes non enregistrées sur la plateforme
. contrôler la présence à la réunion via xref:_registration_code[codes d'inscription].

== Configuration

=== S'inscrire à une réunion

Lorsqu'un participant clique sur "S'inscrire", on lui demande :

* s'il représente un [groupe].
* s'il veut montrer publiquement qu'il assiste à la réunion.

image::components/meetings/registrations/frontend_confirm.png[Join a meeting confirm modal]

=== Formulaire d'inscription

Cette fonctionnalité vous permet de définir un formulaire personnalisé comme le composant xref:admin:components/surveys.adoc[Surveys] pour demander aux participants
des informations d'inscription.

Pour créer un formulaire d'inscription, cocher le "Formulaire d'inscription activé" sur la page des inscriptions, puis cliquer sur "Enregistrer".
Ensuite, il est possibler cliquer sur "Formulaire d'inscription" en haut à droite de la page.

=== Exporter tout

Vous pouvez exporte les enregistrements dans plusieurs formats : CSV, JSON et Excel.

Les données exportées contiennent ces champs :

* id : l'identifiant d'enregistrement
* code : le xref:_registration_code[code d'inscription] (si cette fonctionnalité est activée)
* utilisateur/nom : le nom de l'utilisateur·ice
* utilisateur/email: l'email de l'utilisateur·ice
* user/user_group: le groupe de l'utilisateur·ice s'iel à sélection l'option de représenter le groupe lors de l'inscription

=== Invitations

Cette fonctionnalité permet d'inviter des participant·e·x·s à une réunion. Iels peuvent être des participant·e·x·s enregistré·e·x·s ou des participant·e·x·s
absent·e·x·s de la plateforme.

image::components/meetings/registrations/invite_attendee.png[Invite attendee for registration in a meeting]

=== Code d'enregistrement

Cette fonctionnalité permet de vérifier si unx participant·e·x est bien inscrit·e·x à la réunion.

Après l’inscription, lae participant·e·x reçoit un code d’inscription dans l’e-mail de confirmation et sur la page de la réunion.

image:components/meetings/registrations/frontend_w_code.png[Code d'enregistrement dans une interface de réunion].

La validation du code est en attente. Cela signifie que lors de la participation à une réunion, lae participant·e·x doit donner le code à unx organisateur·ice
pour le valider.
L'organisateur·ice saisit ensuite le code dans le champ "Valider le code d'inscription", pour confirmer qu'iel existe dans la base de données.

image:components/meetings/registrations/backend_w_registration_code.png[Formulaire de code d'inscription dans une réunion]

Une fois le code vérifié, lae participant·e·x est considéré·e·x comme ayant participé à la réunion et le code est marqué comme validé.

image:components/meetings/registrations/frontend_confirmed_w_code.png[Code d'enregistrement dans une interface de réunion]
