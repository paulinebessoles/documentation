= Les budgets

== En bref

Il est important de comprendre que Budget n'est pas seulement le nom de la fonctionnalité, mais qu'il est aussi une entité de la fonctionnalité.
En effet, une même fonctionnalité budgétaire peut contenir plusieurs budgets.

image::components/budgets/multiple_budgets_front.png[Budgets list in frontend]

Cela permet de créer des workflows complexes et de répondre à plusieurs cas d'utilisation.

== Configuration

Une fois la fonctionnalité Budget créée, elle vous redirige vers la liste des budgets. Là, il est possible de créer et de gérer les budgets de
votre fonctionnalité. Selon le processus à organiser souhaité, il est possible de choisir :

* De créer un budget unique, où tous les projets à voter sont affichés.
* De créer des budgets multiples, avec des projets dédiés à l'intérieur de chacun d'eux.

La première chose à faire est de créer le premier budget, en cliquant sur "Nouveau budget".

image::components/budgets/new_budget_button.png[New budget button]

=== Création d'un budget

image::components/budgets/new_budget.png[New budget form]

.Nouveau formulaire de budget
[cols="20h,10h,~"]
|===
|Champ |Type |Description

|Titre
|Obligatoire
|Quel est le titre de ce budget. Par exemple, « District A »

|Rang d'affichage
|Facultatif
|L'ordre dans lequel il se trouve par rapport aux autres budgets. Avoir un nombre inférieur signifie un rang d'affichage plus élevé.
Par exemple, s'il y deux budgets, un "Budget a" de poids "10" et un "Budget b" de poids "0", alors le "Budget b" apparaitra en premier.

|Description
|Facultatif
|La description de ce budget, visible seulement par les participants, dans le cas où plusieurs budgets existent.

|Budget total
|Obligatoire
|La somme d'argent totale disponible pour ce budget.
|===

=== Gérer les budgets

image::components/budgets/manage_budgets.png[Manage budgets table]

Sur les budgets dans la liste de l'administration, les administrateurs peuvent voir :

* Nom : Nom du budget.
* Budget total : La totalité du budget alloué au budget.
* Nombre de projets : Le nombre de projets inclus dans chaque budget.

Cette page permet également de voir les votes passés et en suspens que les différents projets ont reçus pour tous les budgets.

Il est possible d'exporter tous les projets de la fonctionnalité aux formats CSV, JSON et Excel.

=== Flux de travail

Une des caractéristiques du volet Budget est la capacité de définir vos propres flux de travail.

Ceci signifie qu'il est possible de définir plusieurs budgets, par exemple un budget pour chaque quartier d'une ville, ce qui défini les règles selon lesquelles les personnes peuvent voter ces Budgets. Trois workflows sont disponibles par défaut :

* Votez en un seul: permet aux participants de voter dans n'importe quel budget, mais dans un seul.
* Votez en tout: permet aux participants de voter dans tous les budgets.
* Voter dans une composante aléatoire : permet aux participants de voter uniquement dans un seul budget, sélectionné au hasard.

Cette fonction peut être étendue dans l'application par un développeur, en développant vos propres workflows. Par exemple pour la
https://www.decidim. arcelona/processes/PressupostosParticipatius[2020/2021 Processus de budgétisation participative à Decidim Barcelona],
les gens pouvaient voter dans deux districts : celui où ils vivaient et celui de leur choix. Ceci a été défini dans la
https://github.com/AjuntamentdeBarcelona/decidim-barcelona/blob/743be2a210e8745d2dc443166642b285aed2b356/lib/budgets_workflow_pam2021.rb[application decidim-barcelona elle-même].

Plus d'informations à propos des Workflows sur [lien].

NOTE: Vous voulez seulement avoir un vote sans beaucoup de complications ? Vous aimez un vote à l'échelle de la ville? Ne vous inquiétez pas : si un seul Budget est défini, les personnes seront redirigées directement sur la liste des projets.
