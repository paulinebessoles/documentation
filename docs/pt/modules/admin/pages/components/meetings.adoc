= Encontros

== Understand the basics

The Meetings component allows administrators to organize, advertise, deal with registration and wrap up in-person or online meetings.
Depending on the component configuration, meetings can be created only by administrators, or also by participants.

In this documentation section, you can find how to configure and manage all of those items:

* xref:admin:components/meetings/meetings.adoc[Meetings creation]: Create official or participant meetings.
* xref:admin:components/meetings/registrations.adoc[Registrations]: Manage how participants can register to attend the meetings.
* xref:admin:components/meetings/agenda.adoc[Agenda]: Announce the meetings topics to be discussed.
* xref:admin:components/meetings/polls.adoc[Polls]: Enable quick voting during an online meeting.
* xref:admin:components/meetings/close.adoc[Close]: Add the final report and minutes of the meeting.

== Configuração

To create a Meetings component, first go to the xref:admin:spaces.adoc[participatory space] where you want to create it.

image::components/meetings/component_meeting.png[Add Meetings component button]

Then:

. Click on the "Components" item in the secondary menu
. Click the "Add component" button
. Select "Meetings"
. Fill the component creation form

You can still update this form and the configuration after creating the component, by clicking the
image:icons/action_configure.png[Configure icon] icon in the components list.

image::components/meetings/component.png[Add a meetings component]

.Adicionar componente - Formulário de Reunião
[cols="20h,10h,~"]
|===
|Campo |tipo |Descrição:

|Nome:
|Obrigatório
|Qual é o nome desse componente.

|Posição do pedido
|Opcional
|The order of the component in relation with the other components. Ter um número menor significa uma prioridade mais alta.
For instance, if you have two components, "Component a" with weight "10" and "Component b" with weight "0"
then "component b" is first when ordering.
|===

.Adicionar componente - Formulário de Reuniões: Configurações globais
[cols="20h,10h,~"]
|===
|Campo |tipo |Descrição:

|Âmbitos ativados
|Opcional
|Verifique se você quer ter filtragem de escopos neste componente.

|Escopo
|Opcional
|Which xref:admin:settings/scopes.adoc[Scopes] does this component belongs to.

|Anúncio
|Opcional
|A general announcement visible in the meetings list in the participant interface.

|Termos de registro padrão
|Opcional
|The registration terms of service that the participant need to accept when registering.

|Comentários ativados
|Opcional
|Whether you want to have comments creation enabled in the meetings.

|Comprimento máximo de comentários
|Opcional
|The characters limit that participants have when making comments. Deixe 0 para o valor padrão.

|Código de registro habilitado
|Opcional
|Check if you want to automatically give registration codes to participants.
Aprenda mais na xref:components/meetings/registrations.adoc[Registrations].

|Permissões de ações podem ser definidas para cada reunião
|Opcional
|Se você quer mais permissões granulares dentro de cada uma das Reuniões.

|Ativar criação de blocos
|Opcional
|Se você quiser um Etherpad embutido nas suas reuniões. Must have the xref:services:etherpad.adoc[Etherpad] integration configured.

|Os participantes podem criar reuniões
|Opcional
|Permite aos participantes criar novas reuniões. Learn more in xref:components/meetings/meetings.adoc#_participants_meetings[Participants meetings].

|Mapas ativados
|Opcional
|Whether you want to show a map with the meetings addresses in the meetings list and pages.
|===

.Adicionar componente - formulário de reuniões: Configurações do passo
[cols="20h,10h,~"]
|===
|Campo |tipo |Descrição:

|Anúncio
|Opcional
|A general announcement visible in the general meetings list for this phase.

|Comentários ativados
|Opcional
|Whether you want to have comments creation enabled in the meetings for this phase.
|===
